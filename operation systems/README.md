# Общие требования к лабораторным
1) Программа должна корректно работать
2) Код должен быть логически структурирован по файлам, а файлы, в свою
очередь, разбиты по папкам. Если в задаче несколько пунктов и подпунктов,
каждый должен быть реализован в своем отдельном файле
3) Именование: функции, типы, переменные, макросы и т. п. Допускается свой
стиль для каждого вида сущностей, но он должен быть последовательным.
4) Однообразные отступы и расстановка скобок.
5) В коде не должно быть магических констант.
6) Проверяйте возвращаемые значения всех функций и системных вызовов,
корректно реагируйте на ошибки.
7) Все ошибки в коде должны быть обработаны.
8) При корректной работе программа завершается с кодом возврата 0.
9) Программа должна собираться.
10) Сборку желательно осуществлять через Makefile.
11) Код должен собираться без ошибок и предупреждений. Обязательные флаги:
-fsanitize=address -fsanitize=leak -Wall -Wextra -Werror. Если мешают - надо
отдельно объяснить почему
12) Все явно выделенные ресурсы должны быть явно освобождены.
13) Не должно быть путей выхода из программы, где ресурсы остаются
неосвобождёнными.
14) Преподаватель вправе потребовать реализовать дополнительную логику или
задать дополнительные вопросы

## Дополнительные требования для заданий на "хорошо" и "отлично" (предыдущие требования также остаются)
15) Разделяйте сложные функции на понятные части: конкретная функция должна
делать конкретное действие.
16) В коде должны быть понятные комментарии, объясняющие логику работы.
17) Сборка через Makefile становится обязательной
18) Программа должна иметь тесты
19) Тесты должны иметь возможность запуска из одной точки(можете написать свой
скрипт, можете использовать готовый фреймворк)
20) Тестов должно быть несколько, на разные сценарии
